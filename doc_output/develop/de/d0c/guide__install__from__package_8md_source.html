1 @page page_guide_install_from_package Build and install Debian package

3 @brief Describe how to build and install a Leosac deb package.
4 
5 Target platform
6 ---------------
7 
8 This guide is intended to be used on Debian Stretch with amd64 architecture or Raspbian Stretch with armv6/armv7 architecture.
9 
10 Dependencies
11 ------------
12 
13 For starters, install these packages:
14 ```
15 apt-get install cmake build-essential git sudo devscripts
16 ```
17 
18 There is a bug in the ODB compiler found in the Debian and Raspbian repos, which prevents Leosac from building.
19 A bug report has been filed. See [#889664](https://bugs.debian.org/cgi-bin/bugreport.cgi?bug=889664).
20 Until this has been resolved and new pacakges are pushed out to the Debian and Raspbian repos, one must use the odb packages in the Leosac bin-resources repo instead.
21 
22 The following steps were performed on a Raspberry PI running Stretch with gcc 6.4 installed.
23 The folder you choose may differ from this example. Choose the appropriate subfolder names which match your architecture and version of gcc.
24 ```
25 git clone https://github.com/leosac/bin-resources
26 cd bin-resources/debian/gcc6.4/armhf
27 sudo dpkg -i *.deb
28 ```
29 
30 There are additional dependencies required to build Leosac, but those will be installed for you by the deb.sh script.
31 
32 Build
33 -----
34 
35 Get the Leosac Debian packaging script and mark it executable:
36 ```
37 wget https://raw.githubusercontent.com/leosac/leosac/develop/deb.sh
38 chmod +x deb.sh
39 ```
40 
41 If you intend to build the very latest code from the develop branch, then simply call deb.sh with no parameters to start the build process:
42 ```
43 ./deb.sh
44 ```
45 
46 If instead you prefer to build the latest release (or some other branch), then use the -b flag to specify the branch, tag, or release to build:
47 ```
48 ./deb.sh -b v0.7.0
49 ```
50 
51 Replace "v0.70" with the tag name of the most recent release, found on the leosac releases page.
52 
53 That's it!
54 When the build completes, it will present the name of the folder, under /tmp, where the newly built package files reside.
55 From the command line, navigate to that folder and install the Leosac debian package file using dpkg -i or gdebi.
